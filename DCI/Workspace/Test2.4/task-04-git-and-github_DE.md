## Fragen

- Frage 1: Was macht der Befehl `git add .`? Erkläre, was "." bedeutet.
git add . fügt alle Änderungen hinzu und bereitet sie für einen Commit vor, ohne die Änderungen zu speichern.

- Frage 2: Was ist der Unterschied zwischen `git pull` und `git clone`?
git pull wird verwendet wenn man eine Änderung herunterladen will um sie in das aktuelle Repository integrieren möchte. git clone lädt ein komplettes Repository aus dem Internet herunter und erstellt eine neue lokale Kopie.

- Frage 3: Welchen Befehl kannst du verwenden, um den aktuell aktiven Branch zu sehen?
git branch

- Frage 4: Wenn du versuchst, `git log` auszuführen, erhältst du einen Fehler: `fatal: not a git repository` - was bedeutet das?
Man ist nicht in einem Git-Repository Verzeichnis.

- Frage 5: Was macht einen Ordner zu einem Git-Repository?
Wenn man git init als Befehl angibt wird der Ordner zu einem Git-Repository.

- Frage 6: Erkläre den Befehl `git checkout -b new-branch`.
Damit wird eine neuen Branch erstellt und man wechselt zu diesem neuen Branch.

- Frage 7: Was ist der Unterschied zwischen einem Fork und einem Clone?
Ein Clone ist eine lokale Kopie eines Repositories auf deinem eigenen Computer.
Und ein Fork erstellt eine Kopie auf einer Plattform wie Z.b GitHub.

- Frage 8: Was ist der Zweck eines Pull Requests?
Damit werden alle Änderungen am Code korrekt vorgenommen,bevor sie endgültig in das Hauptprojekt integriert werden.

- Frage 9: Erkläre die .gitignore Datei.
Die .gitignore-Datei hilft dabei, das Repository sauber zu halten, Konflikte zu vermeiden und die Performance zu verbessern. Z.b pushen wir damit auf GitHub nichts hoch.
